SCATT 解析ツール 使い方

SCATT Expert の射撃データをリアルタイムに別画面で見たり、過去 trace を分析したりするローカルツール。
データは外部送信せず、すべて自分の Mac 内で完結する。

クイックスタート (3 ステップ)

  1. SCATT Expert を起動して、いつも通り射撃を始める
  2. Finder で scatt-gui.command をダブルクリック
  3. 右側にターゲットと最新 trace が表示される。そのまま射撃を続けるだけで OK。新規 trace は自動で描画され、ウィンドウは最前面に固定される
触らずに使える設計: 起動時から Live polling が自動で動き、常に最前面に表示、画面スリープも抑制される。途中で何か押す必要はない。終了したいときはウィンドウを閉じるだけ。

何ができるか

  • SCATT Expert の trace.data を独自復号、各サンプル X / Y / 銃の傾き を取得
  • リアルタイム監視 (SCATT 動作中も並列読み出し)、セッション切替自動追随
  • 本家 SCATT 互換指標: 10a / 10a-0.5 / 10b / 9c / S1 / S2
  • 本家にない補助指標: 周波数 4 帯域(呼吸/心拍由来/力み/全体)、反動詳細、銃の傾き、心拍 / HRV、approach パターン
  • z-score ランキング: 過去 shot の μ±σ で「今回特にダメな指標」を浮かび上がらせる
  • 5 タブ構成: Dashboard / Sessions / Shots / Help / Settings
  • 23 種のグラフ(自由選択、含 ベスト/ワースト比較、コンディションマップ)
  • BLE 心拍: Apple Watch + HeartCast / 胸ベルト対応、shot 時データを extra.db に永続化
  • ローカル NLG フィードバック: ネット不要、shot / session の所見を自動生成
  • 誤反応 shot の自動検出・削除(SCATT 側 DB と同期)
  • CSV / JSON エクスポート(pandas / Excel / R 用)

起動方法

A. Finder からダブルクリック(推奨)

  1. scatt-gui.command をダブルクリック → 即座に Live モードで GUI 起動
  2. 初回のみ "開発元未確認" の警告が出る場合あり: Finder で右クリック → 開く → ダイアログで再度 開く
  3. 依存パッケージ (PyQt6, numpy) が無ければ初回自動インストール

B. ターミナルから make

cd /Users/kaitabata/Documents/SCATT
make help         # コマンド一覧
make check        # 環境チェック
make gui          # GUI 起動 (Live モード自動 ON)
make watch        # 監視 JSONL (メタ情報のみ) を出力
make watch-full   # 監視 JSONL (samples 含む) を出力
make install      # 依存パッケージを pip でインストール

C. 細かいオプション

# 起動時に Live モードを止めたい (レビュー専用)
/opt/homebrew/bin/python3.10 scatt_gui.py --no-live

# 特定の trace を初期表示
/opt/homebrew/bin/python3.10 scatt_gui.py --trace 5

# 最前面固定をやめる
/opt/homebrew/bin/python3.10 scatt_gui.py     # (--top は付けない)

# 画面スリープ抑制を切る
/opt/homebrew/bin/python3.10 scatt_gui.py --no-caffeinate

# 別の storage.dat を指定
/opt/homebrew/bin/python3.10 scatt_gui.py --db /path/to/storage.dat

  • 左の trace 一覧: 過去 200 件、新しい順。クリックでリプレイ

コマンドラインで JSONL を流す (scatt-watch)

GUI を使わず、別ツールに繋ぎたい場合は scatt-watch.command または make watch を使う。1 行 = 1 trace の JSON。

$ /opt/homebrew/bin/python3.10 scatt_watch.py --no-samples
# watching from trace_id > 935 (current max)
{"trace_id":936,"session_id":4,"timer_ms":1778559500000,"n_samples":null,
 "session":{"distance":50.0,"caliber":5.6,"sample_rate":120,...}}

サンプル配列まで含めたい場合は --no-samples を外す(出力が大きくなる)。jq でフィルタしたり、別プロセスに pipe して可視化に使える。

データ仕様(復号アルゴリズム)

固定 XOR キー (16 バイト)

e3 00 e9 00 34 85 1d 04 f0 95 c0 70 0e 1e b9 f3

復号フロー

  1. BLOB の先頭 1 バイトが 0x01(バージョンマーカー)であることを確認
  2. out[i] = in[i] ^ in[i-1] ^ key[i % 16](CBC 風 XOR、初期 prev = 0)
  3. 結果の先頭 1 バイトを捨てる
  4. 続く 2 バイト = uint16 BE = qChecksum
  5. 残りを qUncompress(先頭 4B BE = 展開後サイズ、続きは zlib stream)
  6. 展開結果は QDataStream 形式、先頭 15 バイトのヘッダ + N × 12 バイトのサンプル

サンプル構造 (12 バイト)

オフセット型意味
0..3float32 BEX 座標 (mm、ターゲット中心からの水平オフセット)
4..7float32 BEY 座標 (mm、垂直オフセット、+ が上)
8..11float32 BECant 角(銃身ロール、radian)

サンプリングレートは sessions.sample_rate カラム(典型 120Hz)。

並行読み出し検証 (SCATT 起動中)

本ツールは storage.dat を read-only で開き PRAGMA busy_timeout を設定するため、SCATT Expert 動作中でも基本的に並行読み取り可能。実機で問題ないか定量的に確認したい場合は scatt_concurrency_test.py を使う。

手順

  1. SCATT Expert を起動し、射撃シミュレーションまたは記録モードに入る
  2. 別ターミナルで以下を実行(デフォルト 60 秒計測): 
    cd /Users/kaitabata/Documents/SCATT
make concurrency-test

# または直接
/opt/homebrew/bin/python3.10 scatt_concurrency_test.py
  3. SCATT 側で 射撃 / トレース生成 / セッション切替などの操作を行う
  4. 60 秒経過 or Ctrl+C で集計表示

計測内容と読み方

項目意味合格目安
polls0.1 秒間隔の SELECT 試行回数— (時間 × 10)
new traces seen計測中に検出された新規 trace_id 数SCATT 側の動作回数と一致
busy errorsSQLITE_BUSY 発生数0 が理想、polls の 1% 未満なら実用可
other errorsその他 SQLite エラー0
query latency p50/p95/p99SELECT の応答時間p99 が数十 ms 以下なら GUI 体感問題なし
trace_id gap warning取りこぼし or 内部削除の検出無いことが望ましい

オプション

# 計測時間と頻度を変える
/opt/homebrew/bin/python3.10 scatt_concurrency_test.py --duration 120 --interval 0.2

# busy_timeout を伸ばす(SCATT 書き込みが長い場合)
/opt/homebrew/bin/python3.10 scatt_concurrency_test.py --busy-ms 5000

busy が出た場合の調整

  • polling 間隔を長く: --interval 0.5 以上(GUI/watch も同様に --interval 0.5)
  • busy_timeout を大きく: --busy-ms 5000
  • SCATT 側の書き込みパターン確認: 大きな BLOB 書き込み中に長時間ロックされている場合あり
ドライラン例: SCATT 未起動の状態で 3 秒走らせたところ、busy 0 / レイテンシ p99=0.4ms。書き込みが無ければレイテンシは負荷ゼロ。

トラブルシュート

"開発元が未確認" で .command が開けない

Finder で右クリック → 開く → ダイアログで再度 開く。

"db not found" と出る

SCATT Expert を一度起動し、何か trace を作ってから再実行。または --db PATH でパス指定。

"ModuleNotFoundError: No module named 'PyQt6'"

make install

または別の Python を使っているなら make gui PYTHON=/path/to/python3 で上書き。

Live で何も流れてこない

  • SCATT Expert を起動して射撃中であることを確認
  • make watch で別途 JSONL に流れるか確認
  • make check で DB のパスが見えているか確認

心拍 (Apple Watch / 胸ベルト)

本ソフトは BLE Heart Rate Profile (0x180D) で心拍を受信する。Mac とのペアリングは 不要。デバイス側でブロードキャストを ON にするだけ。

Apple Watch + HeartCast の場合(検証済み構成)

  1. App Store で HeartCast を購入(Watch アプリ + iPhone コンパニオンアプリの両方が入る)
  2. iPhone で HeartCast を起動 → Apple Watch との連携を許可
  3. iPhone 側で "Start broadcasting" を押す
    ※ iPhone 側がブロードキャスタになって、Mac は iPhone から心拍を受信する構成が安定。Watch から直接ではない。
  4. Watch アプリで心拍計測が動いていることを確認(Workout モード推奨)
  5. Mac 側のターミナルで make ble-scan 実行 → "★ Heart Rate" マークの行があれば成功
  6. GUI を起動 → ToolBar の "HR Start" ボタンを押す → ライブ表示開始
iPhone を経由するメリット: Watch 単体だとアプリ表示中しかブロードキャストできないが、iPhone を経由するとバックグラウンドでも安定して送信される。射撃中は Watch を見ない・iPhone も触らない、という運用に向く。

胸ベルト(Polar H10 / Wahoo TICKR 等)の場合

  1. 胸ベルトを装着して動作中にする(緑 LED 点灯など)
  2. Mac 側で make ble-scan → ベルトが見えれば OK
  3. GUI で "HR Start"

確認用コマンド

# BLE 心拍デバイスを 10 秒スキャン
make ble-scan

macOS Bluetooth 権限

初回 bleak 実行時に macOS が 「Python が Bluetooth へのアクセスを求めています」ダイアログを出す。許可しないと検出できない。あとから変更する場合は
システム設定 → プライバシーとセキュリティ → Bluetooth でターミナル/Python を許可。

表示・分析への活用

  • ToolBar: 現在 HR と直近 30 秒の RMSSD (HRV 指標) をライブ表示
  • Dashboard 指標表: shot 受信時の HR、RMSSD を z-score 比較に含めて色付け
  • Heart rate over time グラフ: ダッシュボードのグラフ枠で選択可能
  • Shots テーブル: shot 単位の HR を一覧化(今後追加予定)
注意: 心拍データは 本ソフトを起動してから観測した shot にのみ 紐づく(SCATT 側の DB には心拍は保存されないため、過去 shot は HR 不明)。観測した shot が増えていくと μ±σ 比較が機能し始める。

今後の拡張候補

  • 手のふるえ・呼吸の FFT スペクトログラム(8-12Hz、0.2-0.5Hz)
  • Pre-trigger 1/2/3 秒の安定度推移(R95、SD)
  • Hold / Aim / Trigger 自動分解(速度閾値ベース)
  • ショット間の重心ドリフト × Cant 解析
  • Apple Watch 心拍 (BLE Heart Rate Profile) の連携
  • 銃身姿勢 IMU (yaw / pitch) 連携
  • 過去ベストショットのゴースト軌跡
  • Claude API による自動コーチング所見生成

SCATT Expert はサードパーティ製品。本ツールはローカルに保存された自分の射撃データを 読み取るだけ で、書き込みやネットワーク送信は行わない。
復号キーとアルゴリズムは SCATT Expert v20.05.31 (Qt 5.9.9, macOS x86_64) で確認。
